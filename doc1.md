CRM UI Tips and Troubleshooting

These tips were learned through 25+ build sessions and will save
significant time when working in the CRM interface. Every team member
who touches the CRM should read this section.

Workflow Builder

- The workflow builder runs inside a cross-origin iframe from
  leadconnectorhq.com. Standard accessibility tools, DOM access, and
  JavaScript do not work inside this iframe.

- The \'+\' buttons on connection lines are invisible by default. Hover
  over the line first to reveal the blue circle. Zoom canvas to 132% or
  higher for reliable clicking. Fallback: use the \'+ Add\' button in
  the top right.

- \'Recent Actions\' is the fastest way to add common actions (Send
  Email, Send SMS, Wait, Add Contact Tag). Warning: Recent Actions
  reorders based on last used, so always verify you selected the right
  action.

- If Recent Actions will not expand, use the search field instead.

- NEVER click \'Cancel\' or \'Discard\' on the unsaved changes dialog.
  This can wipe an entire workflow. ALWAYS click \'Save\' or \'Don\'t
  Leave\'.

- The Draft/Publish toggle is in the top-right of the workflow builder
  header. It may be hidden if the window is too narrow. Resize to 1280px
  or wider.

- Double-clicking an action node resets it. Always single-click only.

- Do NOT use Ctrl+A in a message area because it selects the entire
  page, not just the text field.

Trigger Configuration

- Pipeline Stage Changed triggers need TWO filters: set \'In pipeline\'
  first, then \'Pipeline stage\' becomes available.

- Filter dropdown bug: if no options render, scroll down within the
  dropdown area. Options sometimes render below the visible area.

- For narrow window bugs: widen to 1280px or wider.

- Save Trigger button can be cut off at the bottom. Resize window taller
  (1400x900) to reveal it, or scroll down within the trigger panel.

- Call Details triggers: use \'busy\' + \'no-answer\' for missed calls,
  combined with \'Call Direction = Incoming\'.

Email and SMS Actions

- ALWAYS fill the \'From Email\' field when \'From Name\' is set.
  Leaving it empty causes a silent validation failure.

- Use {{location.email}} for the From Email. Type it directly if the tag
  picker fails.

- Tab key navigates through form fields.

- SMS: Verify character count updates before saving (there is an empty
  body bug). If text will not type: cancel, delete the action, navigate
  away and back, then re-add a fresh SMS action at a lower zoom level.

- Internal Notification SMS type has NO custom message body. Always use
  the Email type for internal notifications.

- Rich text message body bug: text may show 0 characters. Triple-click
  to select all, then retype.

- Save Action: type message, verify character count, click Save Action
  once. If the panel stays open, Tab away. A blue dot on the node means
  it saved.

Tag Creation

- Settings \> Tags \> \'+ New Tag\' \> type name \> click \'Create\'

- The form_input MCP tool is more reliable than typing for tag dialogs.

- Always verify tag count after batch creation.

Pipeline Creation

- Settings \> Opportunities & Pipelines \> Pipelines tab

- Default stages can be renamed by triple-clicking the stage name.

Wait Actions

- Only supports minutes, hours, and days (no months option).

- Convert months to days: 1 month = 30 days, 3 months = 90 days, 11
  months = 334 days.

- After the config panel opens, click the ACTION NAME then press Tab to
  reveal the hidden \'WAIT FOR\' fields.

- To change the dropdown to \'days\': click the dropdown, press
  ArrowDown TWICE, then click \'days\'.

Funnel and Page Builder

- Sites \> Funnels \> click funnel name \> click \'Edit\' on the step.

- Add sections first (1 Column), then elements inside sections.

- Form redirect is set in the PAGE BUILDER right panel (not the form
  builder).

- \'Go to Next step\' redirect can be stubborn. Working method: click
  dropdown to open, press ArrowDown TWICE, press Enter to select.

- AI Builder popup: dismiss with \'Got it\', X, or Escape.

- SEO & AI panel auto-appears after saving pages. Dismiss with X.

Form Builder

- DRAG elements from the Form Element panel (clicking does not add
  them).

- Quick Add tab only has standard contact fields.

- Custom fields appear in \'Add Object Fields\' tab after creation in
  Settings \> Custom Fields.

- Fields already on the form appear grayed out in the panel.

- Forms auto-save (no manual save needed).

- To delete T&C / consent checkbox elements: click the back arrow in the
  right settings panel to close it, THEN the gear/trash icons appear on
  the element. Click trash to delete the entire T&C block at once.

Custom Fields

- Settings \> Custom Fields \> \'+ Add Field\'

- To EDIT an existing field: Folders tab \> click folder name \>
  \'\...\' menu \> Edit (the flat list view has no edit button).

Dashboard Builder

- Use the search field in the Add New Widget panel to find widgets
  faster.

- After adding all widgets: close panel \> \'Save Changes\'.

- Privacy check: three-dot menu \> \'Manage permissions\' \> verify
  \'Private Dashboard\' toggle is OFF (required for snapshots).

Calendar Creation

- Settings \> Calendar Settings

- Prerequisite: Team member must be assigned at the AGENCY level first.

- CRITICAL: Click \'Confirm\' on the creation dialog BEFORE editing
  advanced settings. The calendar is NOT saved until confirmed.

- Keep viewport at 1280x800 (wider viewports can cause freezing).

- Settings appear to auto-save on section navigation.

Switching Sub-Accounts

- Use the sub-account list URL in your CRM dashboard

- Search \> click sub-account \> \'Switch to Sub-Account\'

- The sidebar dropdown method is unreliable.

General Rules

- Window width MUST be 1280px or wider at all times.

- ALWAYS save before navigating away from any builder page.

- API works for READ operations only. Use the UI for all creation tasks.

How to Create a Snapshot

1.  Navigate to Agency \> Account Snapshots in the sidebar (NOT the
    sub-account three-dot menu).

2.  Click \'Create Snapshot\'.

3.  Enter a name (format: \'ALE - \[Module\] v\[X.X\] - \[Month
    Year\]\').

4.  Select the source sub-account.

5.  Click Next, then check ALL assets.

6.  Click Create. Wait for the snapshot to build.

How to Deploy a Snapshot to a New Client

1.  Go to Sub-Accounts and create a new sub-account for the client.

2.  During creation, select the appropriate snapshot as the template.

3.  Fill in the client\'s details: name, email, phone, address,
    timezone.

4.  Note: Country field requires typing \'United States\' into the
    search box inside the dropdown.

5.  After creation, assign the team member at the agency level (required
    for calendars to work).

6.  Run through the onboarding checklist with the client.
